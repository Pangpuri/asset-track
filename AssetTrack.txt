ชื่อโปรเจค AssetTrack

เครื่องมือที่ใช้ Next.js 15 การทำงานแบบ Webapp ใช้ลิงก์เพื่อเข้าถึงได้หลายอุปกรณ์แบบอิสระ 

เทคโนโลยีหลัก
- Next.js 15 (App Router)
- Drizzle ORM
- Supabase (PostgreSQL)
- Tailwind CSS v4
- Better Auth (Authentication System - Schema Defined)
- html5-qrcode (Mobile Camera Scanner)
- Lucide React (Icons)
- jspdf / html2canvas (Export Report)

ระบบแบ่งออกเป็น 8 ส่วนหลัก:

1. ชั้นจัดการ QR (QR Management)
- เพิ่มข้อมูลจาก Dashboard
- ระบบ Generate QR Code ทั้งแบบรายชิ้นและแบบล็อตใหญ่ (Bulk Print)

2. ชั้นการแสดงผล (Display Layer)
- เมื่อสแกน QR จะนำไปยังหน้าข้อมูลอุปกรณ์โดยตรงในรูปแบบที่สวยงาม
- รองรับการเข้าถึงผ่านลิงก์ URL ได้ทันที

3. ชั้นสิทธิ์การใช้งาน (Authorization)
- บุคคลทั่วไป: ดูข้อมูลผ่านหน้า Public Track (Read-Only) ในรูปแบบ IG Style และสแกน QR Code ได้ แต่ไม่สามารถแก้ไขข้อมูลได้
- หน้า Public Track: แสดงข้อมูลครบถ้วน พร้อมระบบแจ้งเตือน "หากอุปกรณ์มีปัญหา โปรดแจ้งฝ่าย MIS"
- Admin (MIS Staff): เข้าถึง Dashboard เพื่อจัดการข้อมูลทั้งหมด, ปรับสถานะ, แก้ไขรายละเอียด, ลบข้อมูล, ดูรายงาน และจัดการบัญชีผู้ใช้งาน

4. ชั้นข้อมูลอุปกรณ์ (Data Layer)
- ข้อมูลพื้นฐาน: รหัส (UUID/P-ID), ยี่ห้อ, รุ่น, ซีเรียลนัมเบอร์, ระยะประกัน
- สถานะวงจรชีวิต: ปกติ (Active), ส่งซ่อม (Repair), สูญหาย (Lost), จำหน่ายออก (Retired)
- ประวัติการทำงาน (Logs): จุดติดตั้ง, ชื่อผู้รับ-ส่งมอบ, วันเวลาที่ทำรายการ

5. ส่วนติดต่อผู้ใช้ (User Interface)
- ดีไซน์สไตล์ Modern IT Dashboard (Zinc/Indigo/Purple)
- หน้า User View ออกแบบในสไตล์ Instagram (IG Style) เพื่อความสวยงามและใช้งานง่าย
- รองรับ Responsive 100% ใช้งานได้ดีทั้ง Mobile (Scanner) และ Desktop

6. ระบบติดตามสถานะ (Lifecycle Tracking)
- ติดตามอุปกรณ์ตั้งแต่รับเข้า จนถึงจำหน่ายออก
- บันทึกประวัติการเปลี่ยนแปลงสถานะอัตโนมัติ

7. การออกรายงานและเอกสาร (Reporting)
- Export ข้อมูลเป็นไฟล์ CSV
- พิมพ์ใบรายการหรือสติกเกอร์ QR เป็น PDF

8. ระบบสมาชิกและการรักษาความปลอดภัย (Auth & Security)
- ใช้ Better Auth สำหรับระบบยืนยันตัวตน (Email & Password)
- Middleware ป้องกันส่วน Dashboard และ Admin API ทั้งหมด
- ระบบจัดการเจ้าหน้าที่ (Staff Management) สำหรับเพิ่ม/ดูรายชื่อทีม MIS ที่มีสิทธิ์เข้าถึงระบบ
